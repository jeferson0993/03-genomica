# Pipeline de Genômica: A Primeira Camada de Processamento Ômico

Se o Projeto 1 coleta os dados e o Projeto 2 os organiza, é no Projeto 3 que a plataforma começa, de fato, a gerar conhecimento biológico. A genômica — o estudo do genoma em sua totalidade — é a disciplina que mais rapidamente avançou nas últimas décadas, impulsionada pela queda vertiginosa dos custos de sequenciamento. Sequenciar um genoma humano completo, que em 2003 custava três bilhões de dólares, hoje pode custar menos de mil. No entanto, esse barateamento trouxe consigo um desafio proporcional: o volume de dados gerados por um único sequenciador ultrapassa a capacidade de análise manual de qualquer pesquisador. É nesse gargalo computacional que o Pipeline de Genômica se insere.

O pipeline segue as GATK Best Practices, o padrão-ouro estabelecido pelo Broad Institute para chamada de variantes germinativas em genomas completos. O fluxo começa com o alinhamento das leituras FASTQ contra o genoma de referência GRCh38 utilizando BWA-MEM, um alinhador que combina velocidade e precisão ao empregar o algoritmo Burrows-Wheeler para indexar o referencial genômico. Em seguida, o GATK4 — mais precisamente o módulo HaplotypeCaller — identifica variantes em cada amostra utilizando um modelo de probabilidade que leva em conta a qualidade das bases, a profundidade de cobertura e o contexto da sequência ao redor de cada posição candidata. O resultado é um arquivo GVCF por amostra, que pode ser combinado posteriormente em um processo de joint genotyping para múltiplas amostras simultâneas.

Entre o alinhamento e a chamada de variantes, contudo, há etapas intermediárias de processamento igualmente cruciais. O MarkDuplicates identifica e marca leituras duplicadas oriundas de artefatos de PCR, evitando que a mesma molécula seja contada múltiplas vezes. O Base Quality Score Recalibration (BQSR) ajusta as pontuações de qualidade das bases com base em padrões de erro observados empiricamente, reduzindo falsos positivos nas regiões de baixa complexidade. Ao final, o VQSR (Variant Quality Score Recalibration) aplica um filtro de qualidade baseado em machine learning, utilizando um conjunto de variantes conhecidas e anotadas como verdadeiras para treinar um modelo que separa variantes reais de artefatos de sequenciamento.

Todo esse processamento é orquestrado pelo Nextflow, que gerencia a execução paralela das etapas, o isolamento em containers Docker e a retomada de execuções interrompidas. Cada etapa utiliza uma imagem oficial: `biocontainers/bwa` para o alinhamento, `broadinstitute/gatk` para as etapas GATK, `staphb/fastqc` e `multiqc/multiqc` para o controle de qualidade. O worker — um container separado da API — executa o Nextflow em segundo plano, enquanto a API FastAPI gerencia o disparo das execuções, o monitoramento de status e a disponibilização dos resultados. Essa separação de responsabilidades garante que a API permaneça responsiva mesmo durante execuções longas, que podem levar dias para um genoma completo.

O controle de qualidade, aliás, não é uma etapa opcional nem meramente decorativa. Seguindo as convenções estabelecidas no manual de boas práticas da plataforma, o pipeline executa FastQC antes e depois de cada etapa crítica, e consolida todos os relatórios em um MultiQC interativo ao final. A inspeção visual da qualidade é parte integrante do fluxo de trabalho, não uma reflexão tardia. Um pesquisador que dispara uma execução recebe, ao final, não apenas um VCF filtrado, mas um conjunto de relatórios que atestam a confiabilidade de cada etapa do processo.

Integrado à plataforma maior, o Pipeline de Genômica lê seus FASTQs de entrada do bucket `raw` do Data Lake e deposita seus resultados — VCFs, GVCFs, relatórios MultiQC, estatísticas de cobertura — nos buckets `processed` e `curated`, registrando cada dataset de saída no catálogo do Projeto 2 com a devida proveniência. Dessa forma, os dados genômicos processados tornam-se imediatamente disponíveis para os módulos downstream: biomarcadores (Projeto 6), redes biológicas (Projeto 7) e inteligência artificial (Projeto 14).

A genômica computacional não é uma ciência nova, mas sua prática ainda enfrenta barreiras significativas de reprodutibilidade e acessibilidade. Ao encapsular as GATK Best Practices em um pipeline containerizado, parametrizável e documentado, o Projeto 3 reduz a barreira de entrada para pesquisadores que precisam processar dados genômicos sem se tornarem especialistas em engenharia de software. O genoma humano tem três bilhões de bases — a plataforma existe para que o pesquisador não precise navegar por cada uma delas manualmente.
